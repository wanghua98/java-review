# 附件文件管理系统概要设计说明书

# 1 引言

## 1\.1 编写目的

本文档用于描述附件文件管理系统的总体设计方案，包括系统架构设计、功能模块设计、数据库设计、接口设计以及关键业务流程设计。

本文档作为系统开发、测试和后期维护的重要依据。

## 1\.2 系统背景

附件文件管理系统主要用于为用户及业务系统提供统一的文件管理能力。

系统支持：

- 普通文件上传 

- 大文件分片上传 

- 文件合并

- 断点续传

- 文件查询

- 文件下载

- 文件预览

- 文件权限管理

- 业务系统附件关联

系统作为独立附件服务，为其他业务系统提供标准化文件服务接口。

# 6 分片上传模块设计（核心）

## 6\.1 设计目标

针对大文件上传过程中：

- 网络不稳定 

- 上传时间过长 

- 上传失败重新开始 

等问题，实现可靠的大文件上传。

支持：

- 文件切片 

- 并发上传 

- 断点续传 

- 秒传优化 

- 文件合并 

## 6\.2 分片上传流程

流程：

```Plain Text
用户选择文件

      |

计算文件信息

      |

生成文件唯一标识

      |

文件切片

      |

上传各个分片

      |

保存上传状态

      |

检查全部分片

      |

合并文件

      |

生成完整文件
```

---

## 6\.3 文件初始化接口

上传开始前：

客户端提交：

```Plain Text
文件名称

文件大小

文件MD5

文件类型

分片数量
```

系统处理：

1. 查询文件是否存在 

2. 创建上传任务 

3. 返回上传编号 

返回：

```Plain Text
{
 "uploadId":"202607091001",
 "chunkSize":5242880,
 "chunkCount":20
}
```

---

## 6\.4 文件分片上传

客户端根据：

chunkSize

进行切割。

例如：

文件：

100MB

切片：

```Plain Text
chunk-1

chunk-2

chunk-3

...

chunk-20
```

上传接口：

```Plain Text
POST /file/chunk/upload
```

参数：

服务器保存：

```Plain Text
临时目录

/upload/temp/uploadId/

       |
       |
       chunk1

       chunk2

       chunk3
```

---

## 6\.5 上传状态查询

用户重新上传时：

查询：

```Plain Text

```

返回：

```Plain Text

 "uploadedChunks":[
 1,
 2,

 ]

}
```

客户端只上传缺少分片。

实现断点续传。

## 6\.6 文件合并

当：

```Plain Text
全部分片上传完成
```

调用：

```Plain Text

```

系统：

```Plain Text

+

chunk2

+

chunk3

...

=

```

合并完成：

生成正式文件记录。

## 6\.7 秒传设计

上传文件前：

计算：

```Plain Text

```

查询：

```Plain Text

```

如果存在：

直接返回文件信息。

流程：

```Plain Text

   |

查询文件

   |

存在？

 /      \

是       否

 |        |

返回     分片上
```

# 9 数据库设计

## 9\.1 数据库设计概述

附件文件管理系统作为一个独立的文件服务平台，需要同时满足**用户文件管理**和**业务系统附件调用**两类场景。

系统数据库主要用于保存：

- 用户基础信息 

- 文件上传任务信息 

- 文件分片上传状态 

- 文件基本信息 

- 文件与业务系统之间的关联关系 

- 文件操作行为记录 

由于系统支持大文件分片上传，因此数据库设计需要区分：

1. **上传过程数据**

2. 用于记录文件上传过程中的任务状态、分片情况。

3. **文件正式数据**

4. 用于保存上传完成后的文件信息，并提供后续查询、下载、预览服务。

系统数据库整体关系如下：

```Plain Text
用户信息

   |

   |

上传任务

   |

   |

文件分片


上传完成


   |

   |

文件信息

   |

   |

业务关联信息



操作日志
```

用于判断所有分片是否上传完成。

例如：

文件：


表示上传完成。
